## Decisions Made

- Chose a daemon-served web client over a second Electron window for the first cut because it keeps the client alive across main app restarts without coupling it to Electron IPC.
- Kept the embedded desktop terminal intact so there is still a rollback path and the existing desktop UX remains available.
- Kept the mobile remote flow unchanged because it already reads from the same backend state and must continue working.
- Dropped `tmux` from the MVP because the daemon already owns the PTY lifecycle and the immediate goal was client separation, not backend multiplexing.
